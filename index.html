How it works

Two-phase pipeline

The LLM never sees raw binary data. Phase 1 extracts clean, typed Parquet with exact column names and value ranges. Phase 2 hands that schema to the LLM — writing pd.read_parquet() against a known schema is trivially reliable.

Phase 1 — Extraction (no LLM)

Capabilities

Everything MAVPose handles for you

From semantic field search to sandboxed execution — focus on the question, not the telemetry plumbing.

🗣️

Plain-English Queries

Describe any flight event. MAVPose resolves the right MAVLink message types automatically via ChromaDB vector embeddings.

📦

Clean Parquet Handoff

The LLM receives a typed, time-aligned Parquet with exact column names, dtypes, and value ranges. No binary guesswork.

🧠

Semantic Field Search

ChromaDB vector store persists to disk. Re-running on the same log skips re-embedding completely.

🔄

Self-Healing Scripts

LLM debugs and rewrites failing scripts up to N times. The fix prompt includes the full schema and traceback.

🛡️

Sandboxed Execution

Generated code runs in an isolated subprocess with a blocked-import denylist and a 30-second timeout.

🔁

Interactive REPL Mode

Omit --prompt to enter a live loop. Query the same log repeatedly without re-parsing. Swap the LLM with a single .env change via OpenRouter.

Installation

Up and running in minutes

Python 3.10+ and an OpenRouter API key are all you need.

# Install the package
pip install mavpose

# Set your API key
set OPENROUTER_API_KEY=your-key-here  # Windows
export OPENROUTER_API_KEY=your-key-here  # macOS / Linux

# Run it
mavpose flight.tlog --prompt "Plot altitude over time"
# 1. Clone
git clone https://github.com/AyushMaria/MAVPose.git && cd MAVPose

# 2. Virtual environment
python -m venv venv && venv\Scripts ctivate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure
cp template.env .env
# Edit .env — add OPENROUTER_API_KEY=your-key-here

# 5. Run
python cli.py flight.tlog --prompt "Plot altitude over time"

⚠️ Note: Never commit .env — it is already in .gitignore. Need a sample log? Download here →

CLI flags

mavpose log_file [options]

  log_file            Path to .tlog / .bin / .log
  -p, --prompt TEXT   Plot request (omit for interactive REPL)
  -r, --retries N     Max self-healing retries (default: 3)
  -v, --verbose       Enable debug logging

Model support

Swap models with one line

MAVPose routes through OpenRouter — any model on the platform works. Set OPENROUTER_MODEL in your .env.

Default

z-ai/glm-5.1
