Pipeline Architecture #

MAVPose uses a strict two-phase pipeline. The LLM never sees raw binary telemetry — it only receives a typed, time-aligned Parquet schema.

1a
Schema Scan + Embeddings parse_mavlink_log() → ChromaDB
↓
1b
Semantic Field Retrieval find_relevant_data_types(prompt)
↓
1c
Full Extraction → Parquet extract_dataframes(msg_types)
↓
2a
LLM Script Generation create_plot(prompt, schema_summary)
↓
2b
Sandboxed Execution + Self-Heal run_script() → plot.png

Quick Start #

Install the package and configure your API key to begin.

bash 
# Install
pip install mavpose

# Configure API key
set OPENROUTER_API_KEY=your-key-here    # Windows
export OPENROUTER_API_KEY=your-key-here  # macOS/Linux

# Run via CLI
mavpose flight.tlog --prompt "Plot altitude over time"

# Or use the Python API
from mavpose.PlotCreator import PlotCreator
creator = PlotCreator(max_retries=3)
creator.set_logfile_name("flight.tlog")
creator.parse_mavlink_log()
types = creator.find_relevant_data_types("altitude over time")
schema = creator.extract_dataframes(types)
creator.create_plot("Plot altitude over time", schema)
creator.run_script()
⚠️

Never commit .env — it is already in .gitignore. Use template.env as a reference and create your own .env from it.

Environment Variables #

All configuration is done via environment variables, typically loaded from a .env file via python-dotenv.

Variable Required Default Description
OPENROUTER_API_KEY Required — Your OpenRouter API key. Used for both LLM inference and embeddings.
OPENROUTER_MODEL Optional z-ai/glm-5.1 The OpenRouter model slug to use for plot generation and self-healing.

CLI — mavpose #

The mavpose command is the headless CLI entry point defined in mavpose/cli.py. It wires the two-phase pipeline end-to-end.

usage 
mavpose log_file [options]

# One-shot
mavpose flight.tlog --prompt "Plot altitude over time"

# With custom retry count and verbose logging
mavpose flight.tlog -p "Show battery voltage" -r 5 -v

# Interactive REPL mode (omit --prompt)
mavpose flight.tlog

Flags Reference

Flag Short Type Default Description
log_file — path Required Path to a MAVLink log file. Accepts .tlog, .bin, .log.
--prompt -p str None Natural-language plot request. Omit to enter interactive REPL mode.
--retries -r int 3 Maximum number of self-healing retries when the generated script fails.
--verbose -v flag False Enable DEBUG-level logging output.
💡

In interactive mode the REPL reuses the parsed schema across queries — re-parsing and re-embedding are skipped on every subsequent prompt. Type quit, exit, or q to stop.

class PlotCreator #

The central orchestrator. Encapsulates the full two-phase pipeline: log extraction → ChromaDB semantic search → LLM script generation → sandboxed execution with self-healing. Defined in mavpose/PlotCreator.py.

init
__init__(self, max_retries: int = 3)
Initialise the PlotCreator, load environment, build LangChain LLM chains.

Loads .env via python-dotenv, reads OPENROUTER_API_KEY and OPENROUTER_MODEL, and constructs two LangChain LCEL chains — a plot-generation chain and a fix chain — both backed by ChatOpenAI pointing at the OpenRouter base URL.

Parameters
NameTypeDefaultDescription
max_retries int 3 Maximum self-healing retries when run_script() encounters an error.
Raises

KeyError — if OPENROUTER_API_KEY is not set in the environment.

Instance attributes set during init
python
creator.logfile_name    = ""          # str — active log path
creator.script_path     = ""          # str — plot.py output path
creator.plot_path       = ""          # str — plot.png output path
creator.parquet_path    = ""          # str — telemetry.parquet path
creator.last_code       = ""          # str — last generated script
creator.message_types   = {}          # dict — schema from schema_only()
creator.db              = None        # Chroma | None
creator.max_retries     = 3           # int
creator.model           = "z-ai/glm-5.1"  # str — active model slug
method
set_logfile_name(self, filename: str) → None
Register a log file and derive all output paths.

Validates the file extension, resolves the absolute path, and sets self.script_path, self.plot_path, self.parquet_path all relative to the log file's directory. Also instantiates the internal LogExtractor.

Must be called before any other method.

Parameters
NameTypeRequiredDescription
filename str Required Path to a MAVLink log file. Must end in .tlog, .bin, or .log.
Raises

ValueError — if the file extension is not in {'.tlog', '.bin', '.log'}.

Example
python
creator.set_logfile_name("logs/2024-06-01.tlog")
# Derived paths (all in logs/):
#   script_path  → logs/plot.py
#   plot_path    → logs/plot.png
#   parquet_path → logs/telemetry.parquet
method
parse_mavlink_log(self) → str
Phase 1a — schema-only scan and ChromaDB embedding build.

Runs a schema-only scan of the MAVLink log — no DataFrames are allocated. Calls LogExtractor.schema_only() to collect message-type metadata, then builds a ChromaDB vector index persisted to <log_dir>/chroma_db/ via OpenAIEmbeddings (using the OpenRouter embeddings endpoint). Subsequent calls to the same log directory skip re-embedding automatically.

Returns

JSON-encoded str of all discovered message types and their fields (pretty-printed with indent=4).

Raises

RuntimeError — if set_logfile_name() has not been called first.

Example output (truncated)
json
{
  "GLOBAL_POSITION_INT": {
    "fields": ["time_boot_ms", "lat", "lon", "alt", "relative_alt"],
    "count": 4821
  },
  "SYS_STATUS": {
    "fields": ["voltage_battery", "current_battery"],
    "count": 960
  }
}
method
find_relevant_data_types(self, human_input: str) → List[str]
Semantic similarity search to resolve relevant MAVLink message types.

Queries the ChromaDB vector store (built by parse_mavlink_log()) with a similarity_search against the user's natural-language prompt. Each stored document is a JSON-encoded single message type; the method extracts the message type names from the retrieved documents and deduplicates them while preserving result order.

Parameters
NameTypeRequiredDescription
human_input str Required The user's natural-language plot request (e.g. "Plot altitude over time").
Returns

List[str] — deduplicated list of MAVLink message type names most semantically relevant to human_input. Preserves order of similarity score.

Raises

RuntimeError — if parse_mavlink_log() has not been called (vector store is None).

Example
python
types = creator.find_relevant_data_types("altitude over time")
# → ['GLOBAL_POSITION_INT', 'GPS_RAW_INT']
method
extract_dataframes(self, msg_types: List[str]) → Dict[str, dict]
Phase 1b — full extraction of message types and export to Parquet.

Runs the full extraction for the given message types via LogExtractor.extract_all(), then calls LogExtractor.export_parquet(msg_types, parquet_path) which:

  • Normalises timestamps into a time_s float64 column (seconds from log start).
  • Tags every row with its msg_type.
  • Concatenates all requested message types into a single time-aligned Parquet file at self.parquet_path.

The returned schema summary is injected directly into the LLM prompt — the LLM never touches the binary log.

Parameters
NameTypeRequiredDescription
msg_types List[str] Required List of MAVLink message type names to extract (e.g. ['GLOBAL_POSITION_INT']). Typically the output of find_relevant_data_types().
Returns

Dict[str, dict] — schema summary keyed by message type. Each value contains rows (int), and columns (dict of col → {dtype, min, max}). Numeric columns include min and max; non-numeric columns only include dtype.

Raises

RuntimeError — if set_logfile_name() has not been called first.

Schema summary shape
python
{
  "GLOBAL_POSITION_INT": {
    "rows": 4821,
    "columns": {
      "time_s": {"dtype": "float64", "min": 0.0,   "max": 623.4},
      "alt":    {"dtype": "float64", "min": 10.2,  "max": 142.7},
      "msg_type":{"dtype": "object"}
    }
  }
}
method
create_plot(self, human_input: str, schema_summary: Dict[str, dict]) → str
Phase 2a — ask the LLM to write a pandas + matplotlib plotting script.

Invokes self._plot_chain (a LangChain LCEL chain with a structured PromptTemplate) with the schema summary, Parquet path, output path, conversation history, and the user's request. The LLM is instructed to return only a fenced ```python``` block. The extracted script is written to self.script_path and cached in self.last_code.

Parameters
NameTypeRequiredDescription
human_input str Required The user's natural-language plot request.
schema_summary Dict[str, dict] Required Schema dict returned by extract_dataframes(). Injected verbatim into the LLM prompt.
Returns

str — the generated Python script. Also written to disk at self.script_path.

LLM Rules enforced via prompt

The LLM is instructed to: read data with pd.read_parquet(parquet_path), filter by msg_type column, use time_s as the X axis (float64 seconds), save the figure to output_file at dpi=400, and never call plt.show(). Prohibited imports: pymavlink, subprocess, os, sys, socket.

method
run_script(self) → tuple
Phase 2b — execute the generated script with sandboxed self-healing.

Runs self.script_path via subprocess.check_output(["python", self.script_path], stderr=STDOUT). On failure, calls attempt_to_fix_script(error_text) and retries up to self.max_retries times. Each fix cycle feeds the stderr and the failing script back to the LLM via self._fix_chain.

Returns

(result, last_code) — a tuple where result is [[(None, (self.plot_path,))]] (plot path wrapper) and last_code is the final script string. If all retries are exhausted, last_code is prefixed with a comment explaining the failure.

Retry behaviour
python
for attempt in range(1, self.max_retries + 1):
    try:
        subprocess.check_output(["python", self.script_path], stderr=STDOUT)
        break  # success — exit loop
    except CalledProcessError as exc:
        if attempt < self.max_retries:
            self.attempt_to_fix_script(exc.output.decode())
method
attempt_to_fix_script(self, error_message: str) → str
Feed the traceback back to the LLM and rewrite the failing script.

Reads the current failing script from self.script_path, reconstructs the schema summary from self._extractor.frames, then invokes self._fix_chain with the script, error text, schema, and Parquet path. The corrected script is written back to self.script_path. Called automatically by run_script() — you rarely need to call this directly.

Parameters
NameTypeRequiredDescription
error_message str Required The stderr or exception text from the failed subprocess run.
Returns

str — the fixed (or best-effort) script. Also written to self.script_path.

Note

If the LLM fix request itself fails (e.g. network error), the original failing script is returned unchanged and the error is logged at ERROR level.

static
extract_code_snippets(text: str) → list
Extract fenced ```python``` blocks from an LLM response string.

Uses re.findall(r"```.*?\n(.*?)```", text, re.DOTALL | re.MULTILINE) to extract all fenced code blocks from the LLM response. Falls back to [text] when no fences are found, so downstream code always receives a non-empty list.

Returns

class LogExtractor #

Headless database-query layer for MAVLink flight logs. Parses .tlog / .bin / .log files into per-message-type pandas DataFrames with a normalised time_s column, and exports a clean Parquet file for LLM consumption. Defined in mavpose/log_extractor.py.

init
__init__(self, log_path: str)
Initialise the extractor and validate the log file exists.

Checks that log_path exists on disk and stores it. Internal state (_frames, _schema, _extracted) is initialised but not populated until extract_all() or schema_only() is called.

Parameters
NameTypeRequiredDescription
log_path str Required Absolute or relative path to a MAVLink .tlog, .bin, or .log file.
Raises

FileNotFoundError — if log_path does not exist.

method
schema_only(self) → Dict[str, dict]
Fast single-pass scan — collects message-type metadata without allocating DataFrames.

Opens the log with pymavlink.mavutil.mavlink_connection() and performs a single pass, recording message type names, field names, Python type strings, and message counts — without building any DataFrame rows. This makes it fast enough to run before embedding without significant memory overhead. Results are stored in self._schema and also returned.

Returns

Dict[str, dict] — mapping of msg_type → {"count": int, "fields": {name: type_str}}. Suitable for building ChromaDB embeddings in PlotCreator._create_embeddings().

Return shape
python
{
  "GLOBAL_POSITION_INT": {
    "count": 4821,
    "fields": {
      "time_boot_ms": "int",
      "lat":          "int",
      "lon":          "int",
      "alt":          "int",
      "relative_alt": "int"
    }
  }
}
method
extract_all(self) → Dict[str, pd.DataFrame]
Full two-pass extraction — builds time-aligned DataFrames for every message type.

Performs a full parse of the MAVLink log in two phases:

  • Pass 1 — collects raw row-dicts per message type.
  • Pass 2 — builds DataFrames, normalises time_s (seconds from first message), casts all numeric columns to float64, drops non-scalar fields (arrays, nested structs), and sorts by time_s.

Timestamps are resolved from MAVLink fields in priority order: time_usec → time_unix_usec → time_us → usec. If none are present, a monotonic fallback (seq × 0.001 s) is used.

Returns

Dict[str, pd.DataFrame] — mapping of msg_type → DataFrame. Every DataFrame has at minimum time_s (float64) and msg_type (str) columns, plus all scalar fields from that message type.

DataFrame column contract
python
df = extractor.frames["GLOBAL_POSITION_INT"]
df.dtypes
# time_s      float64    ← seconds from log start
# msg_type    object     ← always "GLOBAL_POSITION_INT"
# lat         float64
# lon         float64
# alt         float64
# relative_alt float64
method
export_parquet(self, msg_types: List[str], output_path: str) → Dict[str, dict]
Export selected message types to a single time-aligned Parquet file.

Concatenates the DataFrames for the requested msg_types, sorts by time_s, and writes a single Parquet file via pyarrow. Unknown types (not found in the log) are silently skipped with a warning. Builds and returns a schema summary dict with per-column dtype, min, and max — this is injected directly into the LLM prompt by PlotCreator.extract_dataframes().

Parameters
NameTypeRequiredDescription
msg_types List[str] Required MAVLink message type names to include. Types absent from the log are skipped with a warning.
output_path str Required Destination .parquet file path. Parent directories are created automatically.
Returns

Dict[str, dict] — schema summary. Numeric columns include min and max; non-numeric columns include only dtype.

Raises

RuntimeError — if extract_all() has not been called first.
ValueError — if none of the requested msg_types were found in the log.

prop
frames → Dict[str, pd.DataFrame]
Read-only property — returns the extracted frames dict.

Returns self._frames — the dict populated by extract_all(). Returns an empty dict {} until extract_all() has been called. Used by PlotCreator.attempt_to_fix_script() to rebuild the schema summary for the fix prompt.

Returns

Dict[str, pd.DataFrame] — empty until extract_all() is called.

Module Constants #

Top-level constants defined in mavpose/PlotCreator.py and mavpose/safe_executor.py. Override via environment variables where applicable.

Constant Module Value Description
VALID_EXTENSIONS PlotCreator.py {'.tlog', '.bin', '.log'} Accepted MAVLink log file extensions. Checked by set_logfile_name().
OPENROUTER_BASE_URL PlotCreator.py "https://openrouter.ai/api/v1" Base URL for the OpenRouter API, used by both ChatOpenAI and OpenAIEmbeddings.
DEFAULT_MODEL PlotCreator.py "z-ai/glm-5.1" Default LLM model slug. Overridden by OPENROUTER_MODEL env var.
EMBEDDING_MODEL PlotCreator.py "openai/text-embedding-3-small" Embedding model used to build the ChromaDB vector index.
BLOCKED_MODULES safe_executor.py See below Set of module names the generated script is forbidden to import directly.
BLOCKED_MODULES (safe_executor.py)
python
BLOCKED_MODULES = {
    "os", "subprocess", "sys", "shutil",
    "socket", "importlib", "ctypes", "signal",
    "pty", "tty", "termios", "fcntl",
    "pwd", "grp", "resource",
    "multiprocessing", "threading", "concurrent",
    "asyncio", "urllib", "http", "ftplib",
    "smtplib", "pickle", "shelve",
    "dbm", "sqlite3",
}
💡

The import block is enforced via a __import__ hook injected as a preamble into every generated script. The hook checks the caller's __file__ — so stdlib modules loading their own transitive deps (e.g. json → re → enum → sys) are allowed through; only direct imports from the user script are blocked.
